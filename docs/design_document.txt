# Open Sea Tracker Design Document

## Overview

Build a local browser-based HTML, CSS, and JavaScript tracker for a D&D open-sea survival subsystem aboard a damaged ship called the Marrowwind.

The tracker is for the DM to run locally during a session. It should track the ship, crew, supplies, open sea events, planned actions, and turn resolution. The DM and player views are separate HTML pages, with shared styling and split JavaScript files.

The tracker should automate bookkeeping, not replace DM judgment. Manual overrides must always be available.

## Starting File

Primary entry points:

`index.html` redirects to the DM tracker for GitHub Pages.
`open_sea_tracker.html`
`player_view.html`

The player view is opened manually from the DM tracker header after a voyage is started, resumed, or imported. The app should not automatically open player-view popups.

Preserve and improve the current structure incrementally unless the existing structure is unusable.

## Implementation Structure

The DM tracker uses classic browser scripts from `js/` and delegated event handling so it remains build-free. Static and generated controls should use `data-action` for click actions or `data-change-action` for change actions. `js/tracker_gameplay.js` owns the delegated event dispatchers.

DM tracker script order:

1. `js/action_metadata.js`
2. `js/tracker_state.js`
3. `js/tracker_render_setup.js`
4. `js/tracker_render.js`
5. `js/tracker_gameplay.js`
6. `js/tracker_persistence.js`
7. `js/tracker_setup.js`
8. `js/tracker.js`

Responsibilities:

* `js/tracker_state.js` contains shared state, rule tables, and low-level rule helpers.
* `js/tracker_render_setup.js` contains landing/setup rendering.
* `js/tracker_render.js` contains the main DM render dispatcher, tracker-screen rendering, and UI view helpers.
* `js/tracker_gameplay.js` contains DM action handlers, prompts, turn flow, events, meals, and overtime behavior.
* `js/tracker_persistence.js` contains save/load, export/import, player-state publishing, validation, migration, and labels.
* `js/tracker_setup.js` contains setup-mode behavior and setup-to-state creation.
* `js/tracker.js` only boots the tracker after support scripts are loaded.
* `tests/` contains Node tests that load the browser scripts in a VM and exercise validation, migration, import safety, and event-binding coverage.

Implementation conventions:

* Do not add inline `onclick`, `onchange`, or `oninput` attributes.
* Add new button actions through the delegated dispatcher in `js/tracker_gameplay.js`.
* Keep generated markup declarative with `data-*` attributes where practical.
* Run `npm test` after rule, import, migration, or event-binding changes.

## Design Philosophy

* The DM should never fight the UI.
* Manual overrides are always allowed.
* The tracker should automate bookkeeping, not decision making.
* The DM rolls and adjudicates checks unless explicitly automated.
* All calculations should be transparent and visible.
* The entire voyage should be understandable at a glance.
* Keep all characters visible while assigning actions. Dropdowns or compact controls are allowed when they reduce clutter.
* Use large, readable buttons.
* Make it usable on a monitor during a live D&D session.

## Save Import Safety

Import should accept exported tracker JSON and compatible older tracker-state shapes, then migrate them to the current state model.

Import must reject:

* Non-object JSON payloads.
* Oversized files.
* Unsafe object keys such as `__proto__`, `constructor`, or `prototype`.
* Invalid number fields, invalid status fields, malformed crew entries, or malformed state maps.
* Migrated states that still contain impossible crew sizes, non-finite numeric fields, or unknown action ids.

Import should create an undo snapshot before replacing the current state so the DM can recover during the same page session.

## Core Turn Structure

Each ship turn follows this order:

1. Check scripted events or triggers.
2. Roll for Open Sea Event.
3. DM sets each character's action visibly, reviews the party plan, and confirms actions so Labor is spent or gained.
4. Resolve required checks.
5. Update Water Level.
6. Apply Water Level effects.
7. Update Travel Remaining.
8. Advance turn.

Water update formula:

`Water Level = Water Level + Water Ingress - Pumping - Completed Buckets`

Water Level cannot go below 0.

Track these six characters:

* Xander
* Grumbo

Each character needs:

* Labor
* Exhaustion
* Current action

* Manual increase and decrease buttons for Exhaustion

## Action Assignment UI

Do not hide characters behind a single shared action selector.

Each character should appear as a visible row or card with their name and an action control, such as a dropdown or compact button set. The DM should be able to assign all six characters' actions and review the full turn plan before clicking Resolve Turn.

Include a visible Current Turn Plan panel:

* Leopold: selected action
* Delilah: selected action
* Toady: selected action
* Xander: selected action
* Grumbo: selected action
* Tommy: selected action

The DM should be able to clear or change actions before resolving the turn.

## Labor Rules

Characters can safely hold up to 3 Labor.

Whenever a character would gain a 4th Labor point or higher, the tracker should flag that a Constitution save is required.

CON save DCs:

| Labor Reached | CON Save DC |
| ------------: | ----------: |
|             4 |          12 |
|             5 |          14 |
|             6 |          16 |
|             7 |          18 |
|             8 |          20 |

The task still completes even if the save fails.

On failure:

* Add 1 Exhaustion.

The app does not need to roll CON saves automatically. It should clearly prompt the DM with the required DC and allow the DM to mark success or failure.

## Rest Rules

Rest takes 2 full ship turns.

Rest sequence:

* Rest 1/2: no benefit yet.
* Rest 2/2: at end of turn, character chooses one:

  * Recover 2 Labor.
  * Recover 1 Exhaustion.

Resting characters should be visibly marked and unavailable unless manually interrupted.

## Ship Tracking

Track:

* Day
* Turn
* Travel Remaining
* Travel Remaining in ticks, stored internally as 8 travel ticks = 1 day
* Course Meter
* Course State
* Water Level
* Minimum Water Ingress
* Active Leaks
* Water Ingress
* Food
* Water
* Salvaged Timber
* Repair Materials, if implemented

Water Ingress formula:

`Water Ingress = Minimum Water Ingress + Active Leaks`

Current starting values:

* Travel Remaining: 5.5 days / 44 travel ticks
* Course Meter: 12 / True Course
* Water Level: 1
* Minimum Water Ingress: 1
* Active Leaks: 0
* Food: 3.0 crew-days
* Water: 3.0 crew-days

## Course Meter Navigation

The Course Meter is the single navigation meter for the voyage system.

The Course Meter determines the ship's current Course State. The Course State determines the travel change for the turn.

Keep this system simple. Do not add Spot Hazard, Read Sea Conditions, weather subsystems, morale systems, cargo systems, or additional maritime phases.

Course Meter rules:

* Course Meter ranges from 0 to 12.
* Course Meter cannot go below 0.
* Course Meter cannot go above 12.
* At the end of each voyage turn, Course Meter decreases by 1.
* Do not add conditional degradation rolls.
* Travel is stored internally as ticks instead of decimal days.
* Use 8 travel ticks = 1 day.
* The player view should continue displaying Travel Remaining as readable days rounded to the nearest half day.
* The DM view should expose the exact travel tick value.

Course Meter ranges:

| Course Meter | Course State |
| -----------: | ------------ |
|     10 to 12 | True Course  |
|       7 to 9 | Drifting     |
|       4 to 6 | Off Course   |
|       0 to 3 | Lost         |

Travel effects:

| Course State | Travel Change |
| ------------ | ------------- |
| True Course  | Travel Remaining decreases by 0.25 days, which is -2 ticks. |
| Drifting     | Travel Remaining decreases by 0.125 days, which is -1 tick. |
| Off Course   | Travel Remaining changes by 0 days, which is 0 ticks. |
| Lost         | Travel Remaining increases by 0.125 days, which is +1 tick. |

## Ship Systems

Track status for:

* Mast: Working / Broken / Repaired
* Rudder: Working / Broken / Repaired
* Bilge Pump: Working / Jammed
* Net: Ready / Tangled
* Rigging: Intact / Broken

System meanings:

* Mast controls whether the ship can make sail progress.
* Rudder controls whether the ship can be steered.
* Course Meter controls normal Helm travel only when both mast and rudder are functional.

## Water Level Effects

| Water Level | Result                                                                                                         |
| ----------: | -------------------------------------------------------------------------------------------------------------- |
|          1+ | Water in bilge only.                                                                                           |
|          5+ | Water intrudes into cargo hold. Actions below deck take +1 turn to complete.                                   |
|         10+ | Water is waist deep in cargo hold. Actions below deck take +1 additional Labor. Food and water begin to spoil. |
|         15+ | Water is neck deep in cargo hold. Each additional level above 15 adds 1 day to Travel Time.                    |
|          20 | The Marrowwind sinks.                                                                                          |

The tracker should show warnings when Water Level crosses 5, 10, 15, and 20.

## Actions

### 0 Labor Actions

Idle / Watch

* No mechanical effect unless DM adjudicates.

Inventory Food

* Determines exact remaining food.
* Takes 1 turn normally, or 2 turns if the cargo hold is flooded.

Inventory Water

* Determines exact remaining water.
* Takes 1 turn normally, or 2 turns if the cargo hold is flooded.

Inventory Repair Supplies

* Determines exact remaining repair supplies.
* Takes 1 turn normally, or 2 turns if the cargo hold is flooded.

Inventory Cargo

* Determines what cargo remains.
* Takes 1 turn normally, or 2 turns if the cargo hold is flooded.

Navigate / Study Map

* Represents studying the map, checking bearings, comparing charts, plotting the route, or otherwise correcting the ship's course.
* Affects the Course Meter, not Travel Remaining directly.
* Reveals the resulting Course State and Travel Remaining rounded to the nearest 0.5 day.
* Critical failure: Course Meter +0.
* Failure: Course Meter +2.
* Success: Course Meter +4.
* Critical success: Course Meter +5.
* Cap Course Meter at 12 after applying the result.
* The resulting Course State becomes visible to players whether the check succeeds or fails.
* If this action resolves during a turn, skip the normal end-of-turn Course Meter decay for that turn.
* Advantage comes from proficiency with Navigator's Tools or Cartographer's Tools.
* Sailor or Pirate background does not give advantage by default.
* Perception is not a core navigation substitute.

### 1 Labor Actions

Man Helm

* Represents physically steering the ship and making use of the current Course State.
* If mast and rudder are both functional, requires a Survival DC 12 helm check.
* On critical success, apply one Course State better for this turn and improve the Course Meter to the next better Course State.
* On success, apply the current Course State's travel effect.
* On failure, apply one Course State worse for this turn only.
* On critical failure, apply one Course State worse for this turn only and reduce Course Meter by 1.
* If the ship is already Lost, one Course State worse still uses Lost.
* Advantage comes from Sailor background, Pirate background, or proficiency with Water Vehicles.
* Navigator's Tools and Cartographer's Tools do not grant advantage on Man Helm.
* If the mast is broken, do not roll a helm check. The action still costs Labor, but the ship cannot make sail progress and Helm applies no travel tick change.
* If the rudder is broken but the mast is functional, do not roll a helm check. Randomly apply one Course State travel result from True Course, Drifting, Off Course, or Lost. This random result ignores the current Course Meter.
* If both mast and rudder are broken, use the broken mast behavior: no helm check, no random rudder movement, and no travel progress from Helm.

Operate Bilge Pump

* Requires Bilge Pump Working.
* Reduces Water Level by 2 during turn resolution.

Repair Active Leak

* Repairs 1 Active Leak.
* Automatic, no skill check.
* Normally 1 Turn and 1 Labor for 2 Players.
* Requires 1 Repair Supply per leak repaired.
* If Water Level is 5+, takes +1 turn.
* If Water Level is 10+, costs +1 additional Labor.

Repair Bilge Pump

* Repairs Jammed pump.
* Normally 1 turn and 1 Labor.
* Requires 1 Repair Supply.

Repair Rigging

* Repairs Broken rigging.
* Normally 1 turn and 1 Labor.
* Requires 1 Repair Supply.
* Character repairing rigging makes DC 13 Dexterity save or takes 2d6 bludgeoning damage.

Reset Fishing Net

* Makes Net Ready.
* Normally 1 turn and 1 Labor.

Recover Floating Wreckage

* Available only on turns where the Floating Wreckage Open Sea Event is active.
* 1 turn.
* 1 Labor.
* DC 12 Dexterity or Athletics.
* On success, gain salvaged timber.
* Salvaged timber may be consumed at any future time to repair two Active Leaks without Labor.

Salvage Lumber

* Normally 1 turn and 1 Labor for 1 player.
* Adds 2 Repair Supplies when completed.
* Roleplay determines whether the salvage is above deck or below deck.
* If marked below deck, flooding penalties apply as normal.

Bucket Brigade

* Takes 2 turns.
* Costs 1 Labor when started.
* Removes 1 Water Level when completed.

Harpoon Fishing

* Survival check.
* DC 18: +0.5 day Food.
* DC 20: +1 day Food.
* Fisherman background grants advantage.

### 2-Player Actions

Cast Net

* Requires Net Ready.
* Requires 2 players.
* Takes 1 turn.
* Each participant gains 1 Labor.
* Survival DC 15.
* Success: +0.5 days Food.
* After use, Net becomes Tangled.
* If School of Fish event is active, roll with advantage.
* Fisherman background grants advantage to that participant's fishing check.

Harpoon Fishing with Assistance

* Requires 2 players.
* Survival check.
* DC 15: +0.5 day Food.
* DC 18: +1 day Food.
* DC 20: +2 days Food.
* If Large Shadow event is active, roll with advantage.
* Fisherman background grants advantage to that participant's fishing check.

## Major Repairs

Major Repair: Mast or Rudder

* Requires 2 players.
* Takes 2 turns.
* Costs Labor according to the current tracker design.
* Requires 1 Repair Supply.
* If Water Level is 5+, takes +1 turn.
* If Water Level is 10+, costs +1 additional Labor.
* Mast controls sail progress; while broken, Helm cannot make travel progress.
* Rudder controls steering; while broken with a functional mast, Helm movement is random drift rather than a normal Course Meter check.
* Both mast and rudder must be repaired before normal Course Meter Helm behavior resumes.

|  3-5 | Nothing Happens.                                                                                                                                                                               |

| 9-11 | Large Wave. A recent repair or seam below deck comes loose. Add 1 Active Leak.                                                                                                                 |
|   12 | Rigging Snaps. Random creature on deck makes DC 13 Dexterity save or takes 2d6 bludgeoning damage. Rigging can be repaired with 1 turn and 1 Labor.                                            |
|   13 | Pack of Gulls. Run encounter using Swarm of Ravens stat block flavored as gulls. If not dispatched in 3 combat rounds, lose 0.5 days Food.                                                     |

|   16 | School of Fish. The next Cast Net action this turn has advantage. Harpoons gain no benefit.                                                                                                    |
|   17 | Calm Seas. The helmsman recovers 1 Labor instead of gaining 1 while steering this turn.                                                                                                        |
|   18 | Large Shadow. A massive shape circles beneath the ship. It may be targeted with the harpoon, subject to visibility.                                                                            |
|   19 | Favorable Winds. Travel Time -0.25 days if mast and rudder are functional.                                                                                                                     |
|   20 | Phosphorescent Flying Fish & Rain. A light rain begins and a brilliant school of glowing flying fish erupts around the Marrowwind. Gain +1 day Food and +1 day Water.                          |

## Persistent Conditions

Dense Fog

* Lasts 2 turns.
* Visibility reduced to 30 feet.
* Helm checks, fishing checks, attempts to recover floating objects, and ranged attacks against targets not actively engaged in melee are made at disadvantage.
* Targets or objects farther than 30 feet away cannot be targeted normally unless revealed by sound, light, magic, or another clear cue.
* Leopold and Toady's sailor/pirate advantage cancels fog disadvantage on helm checks, so they roll normally in fog.
* School of Fish advantage cancels fog disadvantage on Cast Fishing Net, so the roll is normal.
* Large Shadow advantage cancels fog disadvantage on Harpoon Fishing if the target can be located within range.
* Open Sea Events are still rolled during fog.
* Fog modifies events instead of replacing them.

Broken Rigging

* At the start of each ship turn, before actions are declared, choose a random creature on the top deck.
* That creature makes a DC 13 Dexterity save.
* On failure, it takes 2d6 bludgeoning damage.
* The character repairing the rigging also makes the save.

Jammed Bilge Pump

* Pump cannot be used until repaired.
* Bucket Brigades still work.

Tangled Net

* Cast Net cannot be used until Reset Fishing Net is completed.

## Scripted Events

### Day 1

Standard ship day with 8 turns.

* Turn 0: Breakfast -> Eat.
* Turn 1 starts with Minimum Water Ingress 1.
* Turn 1 starts with 0 Active Leaks.
* Open Sea Events are rolled only every other turn, starting on Turn 2.
* Day 1 Open Sea Event turns are Turn 2, Turn 4, and Turn 6.
* Turn 8: Sehanine's Storm.
* Do not roll an extra Open Sea Event during Sehanine's Storm.
* After Turn 8: Nighttime / required overtime repairs before rest.

Sehanine's Storm is manually handled by the DM in the UI.

During the storm, Mance is knocked overboard by a massive wave after fixing the main sail. At least 2 of his skeletons are burnt to ash. The players must stay up doing overtime work fixing the ship before going to bed. Minimum Water Ingress increases to 2.

### Day 2

* Turn 0: Breakfast -> Eat.
* Turn 7: Nereids.
* Turn 8: Bev.
* After Turn 8: Nighttime -> Rest & Eat.

### Day 3+

* Turn 0: Breakfast → Eat.
* Turn 8: Nighttime → Rest & Eat.

## Nereid Event

Dolphin-riding nereids appear and invite playful roleplay.

They leave sparkling blue trails in the water and encourage the crew to dance, sing, juggle, play music, mimic them, or otherwise lighten up.

Every time a character meaningfully joins the whimsical interaction, reduce that character’s Labor by 1.

The ship still takes on water during this turn.

Travel only progresses if someone still meaningfully mans the helm.

This should be manually handled by the DM in the UI.

## Bev Event

Bev’s pirates disable the ship tactically, not to sink it. Bev wants to plunder the ship, not destroy the cargo by sending it to the bottom.

Bev’s attack should usually do the following:

* Mast becomes Broken.
* Rudder becomes Broken.
* Minimum Water Ingress increases to 3.
* Add Active Leaks, likely +2.
* Encounter consumes 1 ship turn, so Water Ingress still applies.
* If party fights back, pirates retreat after stealing expensive cargo and roughly half the supplies.
* If party does not fight back, pirates steal far more.

Example post-Bev state if the party runs him off after a fight:

* Mast Broken.
* Rudder Broken.
* Active Leaks +2.
* Food roughly halved.
* Water roughly halved.
* Expensive cargo gone.
* Repair supplies reduced but not impossible.
* Water Ingress applies for the turn because no one was pumping during the fight.

If repair supplies are gone, the party can scavenge ship parts manually.

## Scavenge Ship Parts

Optional action if supplies are scarce.

Scavenge Materials

* 1 turn.
* 1 Labor.
* Tear apart nonessential parts of the ship such as furniture, bunks, railings, doors, barrels, or decorative planking.
* Gain 1 Repair Material or equivalent.
* DM may apply a cost, such as cargo loss, reduced comfort, or worsening ship condition.

## Meals and Overnight Rest

Breakfast and dinner each consume up to 0.5 Food and 0.5 Fresh Water.

Food and Water are tracked in crew-days. A meal is available only if both Food and Fresh Water are at least 0.5 before the meal is consumed.

Dinner is resolved at night after Turn 8 and before the crew decides whether to rest or begin Night Overtime. Breakfast is resolved separately before Turn 1.

Overnight recovery depends on which meals were available:

| Meals Available     | Result                                                                 |
| ------------------- | ---------------------------------------------------------------------- |
| Dinner + Breakfast  | Long rest benefits, clear overtime Exhaustion, recover 3 Labor.        |
| Dinner only         | Long rest benefits, clear overtime Exhaustion, recover 1 Labor.        |
| Breakfast only      | Long rest benefits, recover 2 Labor, clear only 1 overtime Exhaustion. |
| Neither             | Recover 1 Labor, clear no overtime Exhaustion, each character makes a Constitution save or gains 1 Exhaustion. |

If neither dinner nor breakfast is available, the Constitution save starts at DC 15. The DC increases by 2 for each consecutive day without meals. There is no maximum DC.

Long rest benefits include:

* Hit points.
* Spell slots.
* Class features.

Labor-based Exhaustion is more persistent and should not automatically recover from a normal night rest.

If the DM determines the rest was meaningfully interrupted or inadequate:

* Adjust Labor or Exhaustion manually on the scoreboard.

## Overtime After Nightfall

If the crew delays sleep and keeps working after nightfall without eating:

* Each additional turn may require a Constitution save.
* Failure adds temporary Exhaustion.
* This Exhaustion may be recovered after eating and completing a long rest.
* Labor-based Exhaustion remains unless specifically recovered.

Tracker behavior:

* At nighttime, dinner is checked first; then the DM can either stop to rest or start Night Overtime.
* While Night Overtime is active, normal ship turns continue.
* Water still ingresses.
* Open Sea Events can still be rolled.
* Actions still resolve normally.
* Recover is not normally available during Night Overtime.
* At the end of each overtime turn, every character with a confirmed action other than Idle / Watch receives a Constitution save prompt.
* Overtime save DC starts at 10 and increases by 2 each overtime turn, to a maximum of 18.
* Failed overtime saves add 1 visible Exhaustion and 1 tracked overtime Exhaustion.
* Ending Night Overtime and resting waits for breakfast, then clears tracked overtime Exhaustion according to the meal table.
* Labor-based Exhaustion is not automatically cleared by ending Night Overtime.

## Resolve Turn Behavior

When Resolve Turn is clicked:

* Apply action effects where automatic.
* Prompt for unresolved rolls.
* Do not silently assume success.
* Apply Labor gains.
* Flag CON save DCs when Labor reaches 4 or higher.
* Track ongoing progress such as Rest 1/2, Bucket Brigade 1/2, and Major Repair progress.
* Apply water update.
* Prevent Water Level from going below 0.
* Show warnings if Water Level crosses 5, 10, 15, or 20.
* Advance the turn unless manually paused.

## Manual Override Requirements

The DM must be able to manually adjust:

* Day.
* Turn.
* Travel Remaining.
* Water Level.
* Minimum Water Ingress.
* Active Leaks.
* Food.
* Water.
* Character Labor.
* Character Exhaustion.
* Ship system statuses.
* Salvaged Timber.
* Current actions.
* Ongoing action progress.

Player-facing knowledge rules:

* Navigate / Study Map reveals the resulting Course State and Travel Remaining rounded to the nearest 0.5 day.
* Water Level is exact only when the players check the Bilge Sounding Rod or when water is visibly in the cargo hold.
* Checking the Bilge Sounding Rod reveals the current Water Level.
* Succeeding on the Bilge Sounding Rod check reveals Total Water Ingress immediately.
* Total Water Ingress also becomes player-facing when the players know the current Water Level for two turns in a row.
* If the current Water Level is not known for the current turn, Total Water Ingress returns to unknown.
* Minimum Water Ingress remains DM-facing.

Scripted scene turns:

* `Force All Idle for Scene` lets the DM handle the current turn as narration/manual resolution instead of normal action resolution.
* It sets crew actions to Idle, removes normal action checks, pauses ongoing work, and logs the scene handling.
* It does not automatically apply specific encounter consequences.

## Visual Design

Use a dark nautical theme.

Prioritize readability over decoration.

Recommended layout:

* Top summary bar: Day, Turn, Travel Remaining, Food, Water.
* Ship panel: Water Level, Minimum Ingress, Active Leaks, Water Ingress, ship systems.
* Crew panel: six character cards.
* Action panel: visible action controls for each character.
* Turn Plan panel: full list of assigned actions.
* Event panel: Open Sea Event table and active conditions.
* Log panel: records turn results and manual changes.

Avoid tiny text. Use high-contrast colors and large buttons suitable for a live table display.
